System Prompt
The normalisation layer converts raw feed data into a unified schema before display and optional backend forwarding. When a backend is connected, normalised records are forwarded automatically. You can also supply a custom transform function.
Unified Event Schema
{
  id:       string,        // unique event id
  type:     string,        // flight | satellite | ship | weather | news | alert | financial | event
  subtype:  string?,       // e.g. "cargo" | "earthquake" | "fx"
  label:    string,        // display name
  body:     string,        // human-readable description
  lat:      number?,       // WGS84 latitude
  lon:      number?,       // WGS84 longitude
  alt:      number?,       // altitude in metres
  speed:    number?,       // speed in knots
  heading:  number?,       // degrees true
  value:    number?,       // e.g. price, magnitude
  change:   number?,       // e.g. % change
  source:   string,        // feed source name
  ts:       number,        // Unix ms timestamp
  ttl:      number?,       // ms until expiry (removes from globe)
  meta:     object?,       // raw source data passthrough
  tags:     string[],      // additional classification tags
  severity: 0|1|2|3?       // 0=info 1=low 2=medium 3=critical
}
Custom Transform (JS)
Optional. Return a normalised event object from raw source data, or null to drop the event.
